## 1 Content

1.1 Fokus

    Das Projekt hat den Fokus Content. Somit sollten alle Entscheidungen Content-gerecht getroffen werden. Ist ein Spieler oder eine Gruppe von Spielern aktiv in einer Content-bezogenen Aktion, so dürfen diese nicht gestört werden.

1.2 Spielerkämpfe

    (a) Das sinnlose Töten von Spielern verursacht ausschließlich Frustration und ist somit untersagt. PvP darf somit nur nach Absprache oder vorhersehbar geschehen.

    (b) Das Stellen von Fallen ist nur bedingt erlaubt. Wird eine Falle gestellt, um einen Spieler, welcher dies nicht beeinflussen kann, zu töten, so ist dies ein Regelverstoß. Dies gilt auch für Spieler, die sich ausgeloggt haben. Fallen sind im Rahmen von Content erlaubt, sofern der Geschädigte zu jedem Zeitpunkt die Situation beenden kann und nicht zwingend getötet wird.

    (c) Das Ausloggen in einem Kampf ist nicht gestattet.

    (d) Das Aktivieren von Totems wird wie das Töten gewertet.

    (e) Die Rückgabe der Ausrüstung eines Spielers muss umgehend nach der Tötung geschehen. Wird der Kampf um einen bestimmten Gegenstand, z. B. das Drachenei, geführt, so darf dieser durch die Tötung erlangte Gegenstand behalten werden.

    (f) Das Töten von wehrlosen Spielern ist untersagt. Findet eine wiederholte Tötung von Spielern statt, kann dies zum permanenten Ausschluss führen.

1.3 Events und Server-Projekte

    (a) Jedem Spieler ist es freigestellt, serverweite Events zu planen und umzusetzen. Dies kann u. U. in Absprache mit der Orga geschehen. Ein Spieler, der ein Event in die Wege leitet, ist im Regelfall der Eventorganisator.

    (b) Spielerkämpfe sind während Events untersagt. Beinhaltet das Event Spielerkämpfe, so nur unter der Moderation eines Eventorganisators.

    (c) Während der Wartezeit sowie Einführung in Events haben sowohl Eventteilnehmende als auch Zuschauende das eigene Mikrofon stumm zu schalten.

    (d) Die Eröffnung von Nether sowie End kann als Event gestaltet werden; sollte dies geschehen, so ist die Orga der Eventorganisator.

1.4 Fraktionen und Gruppen

    (a) Spielern ist es freigestellt, Fraktionen oder Gruppen zu bilden und gemeinsam das Projekt zu gestalten. Für die einfachere Erkennung können sich Fraktionsmitglieder mit einem Prefix kennzeichnen.

    (b) Der Prefix einer bestehenden Fraktion darf nicht kopiert oder imitiert werden. Auf die Farbe des Prefix hat eine Fraktion kein Anrecht.

    (c) Ist eine Fraktion gegründet, so entscheiden deren Mitglieder autonom über die Aufnahme und Entlassung von Spielern.

    (d) Eine Vereinigung von Fraktionen kann frei nach Wunsch beider Fraktionen stattfinden.

    (e) Die Fraktion soll nicht für das Fehlverhalten von Mitgliedern verantwortlich gemacht werden, vorausgesetzt, das Fehlverhalten ist nicht in Absprache mit der Fraktion geschehen.

1.5 Teilnahme

    (a) Die Teilnahme an diesem Projekt ist für Spieler an Content gebunden. Folglich hat die Teilnahme unter der Bedingung stattzufinden, dass die Teilnahme für Content verwertet wird.

    (b) Das Streamen der Teilnahme ist erwünscht.

    (c) Sollte ein Spieler nicht streamen, so muss dieser aufnehmen. Dies ist eine Voraussetzung für eine gute Moderation mit Videobeweisen. Sollte ein Videobeweis nicht möglich sein, so kann die Moderation u. U. nicht stattfinden.

    (d) Fordert die Orga einen Teilnehmer zu einem Videobeweis auf, so muss dieser eine unbearbeitete Aufnahme des genannten Zeitraumes vorlegen. Dies kann in Form eines VODs oder Videos geschehen.

## 2 Unfaire Spielvorteile

2.1 Modifikation vom Spiel und Client

    (a) Die Nutzung von 'Mods' erfolgt nach Whitelist. Sofern eine Modifikation für das Spiel oder den Client nicht explizit erlaubt ist, so kann sie mit einem möglichen Ausschluss aus diesem und zukünftigen Projekten geahndet werden.

    (b) Ein Resourcepack oder Shader, welches die Transparenz, Modelle oder Renderweise von Blöcken maßgeblich verändert, wird als nicht erlaubte Modifikation gewertet.

2.2 Erlaubte Modifikationen

    (a) Eine Modifikation kann erlaubt werden, indem diese von der Orga vor Nutzung geprüft und freigegeben wird. Dies kann durch die dafür vorgesehenen Kommunikationswege in die Wege geleitet werden.

    (b) Eine freigegebene Modifikation kann u. U. nur mit von der Orga vorausgesetzten Bedingungen zur Nutzung freigegeben werden. In solchen Fällen wird die erlaubte Konfiguration und Nutzweise erläutert und bereitgestellt, sofern gegeben.

2.3 Spionage

    (a) Die Übertragungen, VODs sowie Videos eines Spielers dürfen nicht genutzt werden, um Informationen über diesen zu erlangen, welche einen Spielvorteil verursachen. Dies sind u. a. die Koordinaten von Strukturen oder Personen, Ausrüstung von Spielern oder die Erfassung von Bauten.

    (b) Das Abhören von Sprachkanälen, um fraktionsinterne Informationen zu erlangen, wird als unfairer Spielvorteil gewertet. Der Beitritt in einen fraktionsfremden Sprachkanal kann nur erfolgen, sofern dieser deutlich als offen gekennzeichnet ist (z. B. mit dem Suffix 'open' im Namen). Eine Ausnahme ist, wenn der Beitritt nach Aufforderung oder Absprache geschehen ist.

2.4 Nutzung von Spielfehlern

    (a) Die beabsichtigte Nutzung von Spiel- und Serverfehlern ist nicht gestattet. Sollte ein Spielfehler auftreten, so ist dieser umgehend der Orga zu melden.

    (b) Die Duplikation von 'Items' oder 'Entities' ist nicht gestattet. Hiervon ausgeschlossen sind 'TNT-Duper'.

    (c) Die Zerstörung von Bedrock ist grundsätzlich nicht gestattet. Eine Ausnahme bildet die Zerstörung von Bedrock, um die Netherdecke zu erreichen.
    
## 3 Allgemeines

3.1 Items

    (a) Sollten Items verloren gehen, so hat der Spieler kein Anrecht darauf, dass dieses von der Orga ersetzt wird.

    (b) Sollte ein Fall von besonderem Interesse bestehen, so besteht die Möglichkeit, dass die Orga ein Item auffindig macht. Auch hierbei hat der Interessierte kein Anrecht auf die Aushändigung durch die Orga.

    (c) Das Entwenden von Gegenständen aus Behältern, die nicht offensichtlich für die Allgemeinheit bestimmt sind, wird als Diebstahl gewertet und ist verboten. Dies gilt auch für ungesicherte Bereiche.

    (d) Ein Behälter ist dann für die Allgemeinheit bestimmt, wenn dieser mit einem Schild als dieses gekennzeichnet ist. Hat ein Behälter keine Kennzeichnung, so ist davon auszugehen, dass dieser nicht für die Allgemeinheit bestimmt ist.

3.2 Lag

    (a) Wenn die Leistung des Servers durch auf diesem Server existierende Entities oder Bauten beeinflusst wird, so hat die Orga das Recht diese von dem Server zu entfernen.

    (b) Der gezielte Versuch die Leistung des Servers zu beeinflussen wird mit einem Ausschluss von mindestens 48h Dauer bestraft.

    (c) Spielern ist es freigestellt, Chunks zu laden. Hierbei ist zu beachten, dass das Laden einer unverhältnismäßig hohen Menge von Chunks durch einen einzelnen Spieler zu vermeiden ist.

    (d) Das dauerhafte, unbeaufsichtigte Nutzen von Farmen (AFK-Farming) über einen längeren Zeitraum ist zu unterlassen.

3.3 Bauen und Verhalten in der Welt

    (a) Das Griefing, also das mutwillige Zerstören oder Verunstalten von Bauten anderer Spieler, ist strengstens untersagt.

    (b) Das Errichten von Bauten, die anstößig, beleidigend oder diskriminierend sind, ist verboten.

    (c) Das Claimen von Gebieten ist in einem angemessenen Rahmen zu halten. Übermäßig große Gebietsansprüche, die andere Spieler in ihrer Entfaltung stark einschränken, sind mit der Orga abzusprechen.

    (d) Bei Verlassen des Projekts sollten große Farmen oder technische Anlagen entfernt werden, um die Serverleistung nicht dauerhaft zu beeinträchtigen.

    (e) Großflächiges Terraforming, das das Landschaftsbild nachhaltig zerstört, sollte in einem angemessenen Rahmen bleiben.

3.4 Handel

    (a) Betrug beim Handel mit anderen Spielern ist verboten. Vereinbarungen sind einzuhalten.

    (b) Es ist nicht gestattet, Spieler zu einem Handel zu zwingen oder unter Druck zu setzen.

    (c) Das Errichten von Handelszentren oder Shops ist erwünscht und fördert die Server-Wirtschaft.

    (d) Der Handel ist offen. Ein Exklusivrecht auf Handelsgüter ist nicht gegeben.



## 4 Kommunikation und Interaktion

4.1 Grundsatz für Kommunikation

    (a) Die Kommunikation und folglich Interaktion ist mit Würde und Respekt umzusetzen. Niemand darf aufgrund des eigenen Geschlechts, der Herkunft oder Gesinnung benachteiligt werden. Diskriminierung jeglicher Art wird nicht geduldet.

    (b) Das Verbreiten von Falschinformationen wird nicht geduldet.

    (c) Politische Diskurse sind nicht erwünscht; dieses Projekt ist für jeden gedacht, Politik hat in einem Gemeinschaftsprojekt nichts zu suchen.

    (d) Das Spammen von Nachrichten, die exzessive Nutzung von Großbuchstaben sowie Eigenwerbung sind im globalen Chat zu unterlassen.

4.2 Interne Informationen

    (a) Interne Informationen sind solche, die nicht für die Öffentlichkeit bestimmt sind. Diese Informationen dürfen nicht frei geteilt oder weiter verbreitet werden.

    (b) Interaktionen, welche mit der Orga stattfinden, u. a. die Problemlösung bei Konflikten, sind stets interne Informationen.

    (c) Ankündigungen sind in der Regel interne Informationen; eine Ausnahme bilden öffentliche Ankündigungen von Events.

    (d) Dieses Regelwerk, technische Informationen sowie Abstimmungen über die Teilnahme von Spielern werden auch als interne Information gehandhabt. Wenn ein Spieler offiziell aufgenommen wurde, so darf dieser die Aufnahme verkündigen. Ist dies geschehen, ist die Information nicht mehr intern.

4.3 Soundboards

    (a) Die Nutzung eines Soundboards sowie das Abspielen von Audio ist untersagt.

    (b) Die Nutzung eines Soundboards kann bei einem Event durch die Eventorganisation in Erwägung gezogen werden. Sollte dies der Fall sein, so darf nur diese in Rücksprache mit der Orga das Soundboard verwenden.

    (c) Die exzessive Nutzung von Soundboards im Eventkontext ist untersagt.

## 5 Weiteres

5.1 Terms of Service

    Die ToS (Terms of Service) oder EULA (End User License Agreements) von den folgenden Plattformen dürfen unter keinen Umständen verletzt werden und werden mit einem Ausschluss von mindestens 7 Tagen bestraft: Minecraft, YouTube, Twitch, TikTok sowie Instagram.

5.2 Urheberrechtlich geschütztes Material nach §2 UrhG

    UgM (Urheberrechtlich geschütztes Material) darf nicht verbreitet werden. Dies beinhaltet das Abspielen von Musik durch u. a. den Voicechat.

5.3 Host und Organisation (Orga)

    (a) Die dauerhafte Bereitstellung des Servers kann nicht sichergestellt werden. Der Server wird feste Zeiten haben, in welchen dieser aus technischen sowie organisatorischen Gründen nicht verfügbar ist.

    (b) Die Bearbeitung von Anliegen unterliegt keinem besonderen Drang (Eile), d. h., die Bearbeitung von u. a. 2.2a kann u. U. verzögert oder nicht geschehen. Ist dies der Fall, so muss auf den Antrag verzichtet werden.

    (c) Die Entscheidungen der Orga sind immer mit diesem Regelwerk zu rechtfertigen. Ist dies nicht möglich, so dürfen die Entscheidungen angefochten werden. Wurde die Rechtfertigung korrekt nach Regelwerk gestaltet, so ist diese Entscheidung bindend.

    (d) Die Orga darf nicht aktiv am Spielgeschehen teilnehmen. Dies beinhaltet u. a. den Handel sowie eine gemeinsame Fraktion. Die Orga behält sich vor, dass sie abseits des allgemeinen Geschehens auf derselben Welt aktiv sein kann. Sollte dies der Fall sein, dürfen reguläre Spieler, die das Gebiet entdecken, nicht mit dem Gebiet interagieren. Die Orga hat das Gebiet entsprechend zu kennzeichnen.

    (e) Die Orga behält sich vor, in Stichproben die Spieler zu kontrollieren; dies kann auch ohne Grund geschehen.

5.4 Regelwerk

    (a) Dieses Regelwerk ist für jeden Spieler und die Orga verbindlich.

    (b) Für eine Änderung an diesem Regelwerk stimmt die Orga ab; hierbei ist eine 2/3 Mehrheit vorausgesetzt. Alle Teilnehmer haben das Recht, eine Änderung für das Regelwerk vorzuschlagen, wird dieser Vorschlag von der Mehrheit der aktiven Spieler als sinnvoll erachtet, so muss die Orga über diese Änderung abstimmen.

    (c) Wird eine Änderung des Regelwerkes beschlossen, so werden alle Teilnehmer über diese informiert. Um eine weitere Vereinbarung zu gewährleisten, kann eine neue Bestätigung der Regeln von der Orga verlangt werden.

5.5 Strafen und Rahmenbedingungen

    (a) Ein temporärer Ausschluss kann, sofern keine Dauer angegeben wurde, formal bei der Orga aufgehoben werden.

    (b) Hat ein permanenter Ausschluss stattgefunden, so ist dieser auch für folgende Projekte geltend.

    (c) Das Mindestalter für die Teilnahme ist 16.

    (d) Die Teilnahme kann durch eine formale Bewerbung sowie nach Vorschlag durch einen etablierten Teilnehmer in die Wege geleitet werden. Sollte eine Teilnahme plausibel sein, stimmen etablierte Teilnehmer für die Aufnahme ab; hierbei ist eine absolute Mehrheit vorausgesetzt.

    (e) Als 'etablierter Teilnehmer' gilt, wer aktiv am Projekt teilnimmt und nicht negativ durch Regelverstöße aufgefallen ist.
